Table Per Concrete

Steps to use Table Per Concrete in Hibernate Inheritance


1. Suppose we are making an application of Employee data; we need to create a table named EMP with the following parameters:
* Emp Table in DataBase

 


2. Now make another table in database named REGEMP with the following parameters:
* Regemp Table in DataBase

 


3. Make another table named CONEMP with its respective parameters:
* Conemp Table in DataBase

 


4. It is time to create another POJO class named as Employee1.java with parameters int id and String name. Create its setters and getters.
* Employee1.java

 


5. Make another POJO class named as Contract_Employee1.java with parameters pay_per_hour and String duration. Create its setters and getters.
* Contract_Employee1.java

 


6. It is time to create another POJO class named as Regular_Employee1.java with parameters int salary and int bonus. Create its setters and getters.
* Regular_Employee1.java

 


7. Create the servlet for configuring these POJO class' objects. Enter the values and save it to database.
* Servlet.java

 


8. In hibernate1.cfg.xml, add mapping resource by giving the name of mapping xml.
* hibernate1.cfg.xml

 


9. Enter the objects' name and their properties in mapping xml.
* hibernate1.hbm.xml
