Hibernate Naming Queries


Often times, developer like to put HQL string literals scatter all over the Java code, this method is hard to maintaine and look ugly. Fortunately, Hibernate come out a technique called "names queries" , it lets developer to put all HQL into the XML mapping file or via annotation.
The hibernate named query is way to use any query by some meaningful name. It is like using alias names. The Hibernate framework provides the concept of named queries so that application programmer need not to scatter queries to all the java code.
There are two ways to define the named query in hibernate:
1. By annotation
2. By mapping file
Named query definition has two important attributes:
 1. name: Here the name of named query is specified, by which it will be located using hibernate session.
2. query: Here the HQL statement is given ,which we want to execute in database.

Example of naming query with Annotations

1.Create Student.java class with annotations,and for naming query ,we use the annotation @NamedQueries within which we use @NamesQuery.
It has two parameters name,name to recognise the query and query,containing the query to be executed.
